## 一、系统总览

### 1.1 模块接口

```
                    ┌─────────────────────────────┐
              clk ──┤                             ├── hs
              rst ──┤   hdmi_image_display        ├── vs
                    │                             ├── de
                    │   + image_rom_128x72        ├── rgb_r[7:0]
                    │                             ├── rgb_g[7:0]
                    │                             └── rgb_b[7:0]
                    └─────────────────────────────┘
```

| 端口 | 方向 | 位宽 | 说明 |
|------|------|------|------|
| `clk` | I | 1 | 像素时钟，通常 74.25 MHz (720p) |
| `rst` | I | 1 | 异步复位，高有效 |
| `hs` | O | 1 | 行同步信号 |
| `vs` | O | 1 | 场同步信号 |
| `de` | O | 1 | 数据有效信号（高 = 有效像素） |
| `rgb_r` | O | 8 | 红色分量 |
| `rgb_g` | O | 8 | 绿色分量 |
| `rgb_b` | O | 8 | 蓝色分量 |

## 二、HDMI 时序参数（720p 标准）

### 2.1 时序定义

```
         ←──────────────── H_TOTAL (1650) ────────────────→
         ← H_ACTIVE (1280) →← H_FP(110) →← H_SYNC(40) →← H_BP(220) →
    ─────┐                ┌───────────────────────┐                 ┌─────
         └────────────────┘                       └─────────────────┘
             有效像素区        行消隐区（前沿+同步+后沿）
    ─────┐                                                          ┌─────
         │             de = 1                     de = 0             │
         └──────────────────────────────────────────────────────────┘
```

```verilog
parameter H_ACTIVE = 16'd1280;   // 一行有效像素数
parameter H_FP     = 16'd110;    // 行前沿 (Front Porch)
parameter H_SYNC   = 16'd40;     // 行同步脉冲宽度
parameter H_BP     = 16'd220;    // 行后沿 (Back Porch)

// 衍生参数
localparam H_TOTAL = H_ACTIVE + H_FP + H_SYNC + H_BP;  // = 1650
localparam H_START = H_FP + H_SYNC + H_BP;             // = 370 (有效像素起始位置)
```

### 2.2 行扫描流程（`h_cnt` 驱动）

| h_cnt 范围 | 区域 | hs | de | 说明 |
|------------|------|----|----|------|
| 0 ~ 109 | 前沿 | 0 | 0 | 上一行结束后的间隔 |
| 110 ~ 149 | 同步 | 1 | 0 | 行同步脉冲（显示器不显示） |
| 150 ~ 369 | 后沿 | 0 | 0 | 同步后到有效像素前的间隔 |
| **370 ~ 1649** | **有效像素** | **0** | **1** | **显示器显示的区域** |

> `h_cnt` 在 0 ~ 1649 之间循环计数（`H_TOTAL - 1 = 1649`）。

### 2.3 场扫描流程（`v_cnt` 驱动）

| v_cnt 范围 | 区域 | vs | de | 说明 |
|------------|------|----|----|------|
| 0 ~ 4 | 前沿 | 0 | 0 | 上一帧结束后的间隔 |
| 5 ~ 9 | 同步 | 1 | 0 | 场同步脉冲 |
| 10 ~ 29 | 后沿 | 0 | 0 | 同步后到有效行前的间隔 |
| **30 ~ 749** | **有效行** | **0** | **有效行时=1** | **显示器显示的区域** |

> `v_cnt` 在 0 ~ 749 之间循环计数（`V_TOTAL - 1 = 749`）。

## 三、图像显示区域

### 3.1 核心参数

```verilog
localparam IMG_WIDTH   = 128;     // ROM 源图像宽度
localparam IMG_HEIGHT  = 72;      // ROM 源图像高度
localparam SCALE       = 4;       // 缩放因子（2 的幂 → 移位实现除法）

localparam IMG_DISP_W  = IMG_WIDTH  * SCALE;   // = 512（显示宽度）
localparam IMG_DISP_H  = IMG_HEIGHT * SCALE;   // = 288（显示高度）

// 居中偏移量
localparam IMG_OFFSET_X = (H_ACTIVE - IMG_DISP_W) / 2;  // = 384
localparam IMG_OFFSET_Y = (V_ACTIVE - IMG_DISP_H) / 2;  // = 216
```

### 3.2 图像区判定 — `image_area` 信号

```
     active_x:   0 ──────────────── 384 ──────── 896 ─────────────── 1279
                                    ├── IMG_DISP_W=512 ──┤
                                    │                     │
                 ┌──────────────────┼─────────────────────┼──────────────────┐
     active_y:   │     背景色区域    │                     │    背景色区域      │
              0  │                  │                     │                   │
                 │                  │                     │                   │
            216  ├──────────────────┼─────────────────────┼──────────────────┤
                 │                  │                     │                   │
                 │    背景色区域     │    图像区域          │    背景色区域      │
                 │                  │   image_area=1      │                   │
            504  ├──────────────────┼─────────────────────┼──────────────────┤
                 │                  │                     │                   │
                 │    背景色区域     │                     │    背景色区域      │
            719  └──────────────────┴─────────────────────┴──────────────────┘
```

```verilog
assign image_area = (active_x >= IMG_OFFSET_X[11:0]) &&
                    (active_x <  (IMG_OFFSET_X + IMG_DISP_W)) &&
                    (active_y >= IMG_OFFSET_Y[11:0]) &&
                    (active_y <  (IMG_OFFSET_Y + IMG_DISP_H));
```

## 四、ROM 读地址与图像缩放的对应关系

### 4.1 从屏幕坐标反推源图像坐标

图像放大 `SCALE` 倍后，屏幕上每个 `SCALE × SCALE` 的像素块对应源图像的一个像素：

```
源图像 (128×72)                    屏幕显示 (512×288, SCALE=4)
┌────┬────┬───                     ┌──────┬──────┬───
│0,0 │1,0 │...                     │0..3, │4..7, │...
│    │    │                        │0..3  │0..3  │
├────┼────┼───                     ├──────┼──────┼───
│0,1 │1,1 │...        ──4×缩放──▶  │0..3, │4..7, │...
│    │    │                        │4..7  │4..7  │
├────┼────┼───                     ├──────┼──────┼───
│... │... │...                     │...   │...   │...
```

**坐标反推公式：**

$$
\text{image\_x} = \frac{\text{active\_x} - \text{IMG\_OFFSET\_X}}{\text{SCALE}} \qquad
\text{image\_y} = \frac{\text{active\_y} - \text{IMG\_OFFSET\_Y}}{\text{SCALE}}
$$

### 4.2 为什么 SCALE 用 2 的幂

| SCALE | 综合实现 | 资源 |
|-------|---------|------|
| 2, 4, 8, 16... | **右移** (>>1, >>2, >>3, >>4) | 零 LUT |
| 3, 5, 6, 7, 10... | 乘法器 (倒数乘法) | DSP48 或大量 LUT |

> `SCALE=4` 时 `active_x / 4` 综合为 `active_x >> 2`，不消耗任何逻辑资源。

### 4.3 线性地址映射

```verilog
// image_x: 7bit (0~127), image_y: 7bit (0~71)
// image_addr = image_y * 128 + image_x
//            = {image_y, 7'b0} + {7'd0, image_x}
assign image_addr = {image_y, 7'b0} + {7'd0, image_x};
```

`{image_y, 7'b0}` 等价于 `image_y << 7` = `image_y × 128`，行主序线性编址。

### 4.4 ROM 例化

```verilog
image_rom_128x72 u_image_rom_128x72 (
    .clk(clk),
    .addr(rom_addr),      // 14bit 地址 (0 ~ 9215)
    .data(image_pixel)    // 24bit RGB888 像素数据
);
```

- 深度：`128 × 72 = 9216`，地址位宽 14bit
- 位宽：24bit (R[23:16] G[15:8] B[7:0])
- 读延迟：**1 个时钟周期**（`rom_addr` 寄存器 → ROM → `image_pixel`）

## 五、RGB 输出逻辑

### 5.1 三级选择

```verilog
assign rgb_r = de_reg_d0 ? (image_area ? image_pixel[23:16] : BG_R) : 8'h00;
assign rgb_g = de_reg_d0 ? (image_area ? image_pixel[15:8]  : BG_G) : 8'h00;
assign rgb_b = de_reg_d0 ? (image_area ? image_pixel[7:0]   : BG_B) : 8'h00;
```

| de_reg_d0 | image_area | RGB 输出 | 屏幕显示 |
|-----------|------------|----------|---------|
| 0 | X | `8'h00` (黑) | 消隐区（不看） |
| 1 | 0 | `{BG_R, BG_G, BG_B}` | **背景色边框 ← 可见！** |
| 1 | 1 | `image_pixel` | 图像内容 |

### 5.2 背景颜色配置

```verilog
localparam BG_R = 8'h20;   // 深蓝灰底色
localparam BG_G = 8'h20;
localparam BG_B = 8'h60;
```

修改这三个 `localparam` 即可改变背景色：

| 颜色效果 | BG_R | BG_G | BG_B |
|----------|------|------|------|
| 深蓝灰（当前） | `8'h20` | `8'h20` | `8'h60` |
| 纯黑 | `8'h00` | `8'h00` | `8'h00` |
| 纯白 | `8'hFF` | `8'hFF` | `8'hFF` |
| 暗红 | `8'h40` | `8'h00` | `8'h00` |
| 暗绿 | `8'h00` | `8'h40` | `8'h00` |
| 海军蓝 | `8'h00` | `8'h00` | `8'h80` |

## 六、信号时序流水线

### 6.1 延迟关系

```
时钟周期:        N         N+1        N+2        N+3
              ─────     ─────      ─────      ─────

h_cnt/v_cnt   [ 值 ]────▶[ 值 ]─────▶[ 值 ]─────▶[ 值 ]
              │         │          │          │
active_x/y  ──┘         │          │          │
image_area  ────────────┘          │          │
image_x/y   ──────────────────────┘          │
image_addr  ─────────────────────────────────┘
              │         │          │          │
rom_addr     ──[X]──────▶[M]───────▶[M+1]─────▶[M+2]
              │         │          │          │
image_pixel  ───────────▶[ROM[M]]──▶[ROM[M+1]]
              │         │          │          │
de_reg_d0    ───────────▶[1]───────▶[1]───────▶[1]
              │         │          │          │
rgb_r/g/b    消隐期输出  [ROM[M]]──▶[ROM[M+1]]
                         正确显示   正确显示
```

### 6.2 关键点

1. **ROM 读延迟 = 1 周期**：`rom_addr` 在周期 N 锁存 → `image_pixel` 在周期 N+1 有效
2. **de_reg_d0 延迟 = 2 周期**：`video_active`(组合) → `de_reg`(周期 N+1) → `de_reg_d0`(周期 N+2)
3. **image_area 为组合逻辑**：与 `h_cnt`/`v_cnt` 同步变化，无额外延迟
4. 存在 **1 像素的固定偏移**，但在 1280×720 分辨率下不可察觉

## 七、参数调节指南

### 7.1 改变图像大小

修改 `SCALE`（推荐保持 2 的幂）：

| SCALE | 图像显示尺寸 | 左右背景边宽 | 上下背景边高 |
|-------|-------------|-------------|-------------|
| 2 | 256 × 144 | 512 | 288 |
| **4** (默认) | **512 × 288** | **384** | **216** |
| 8 | 1024 × 576 | 128 | 72 |

> 也可直接设 `IMG_DISP_W` / `IMG_DISP_H` 为非整数缩放尺寸，但 `active_x / SCALE` 会综合出除法器。

### 7.2 改变背景颜色

修改第 43-45 行：

```verilog
localparam BG_R = 8'hXX;   // 替换为目标 R 值
localparam BG_G = 8'hXX;   // 替换为目标 G 值
localparam BG_B = 8'hXX;   // 替换为目标 B 值
```

### 7.3 改变分辨率

修改 `parameter` 块即可适配其他 HDMI 分辨率：

| 分辨率 | H_ACTIVE | H_FP | H_SYNC | H_BP | V_ACTIVE | V_FP | V_SYNC | V_BP | 像素时钟 |
|--------|----------|------|--------|------|----------|------|--------|------|----------|
| 720p (当前) | 1280 | 110 | 40 | 220 | 720 | 5 | 5 | 20 | 74.25MHz |
| 1080p | 1920 | 88 | 44 | 148 | 1080 | 4 | 5 | 36 | 148.5MHz |

## 八、常见问题

**Q1: 为什么消隐期输出黑色而不是背景色？**

消隐期（`de=0`）包括同步脉冲区间，HDMI 规范要求此期间 RGB 数据为 0。且消隐期的像素在显示器上不可见，设什么颜色都看不到。

**Q2: 为什么背景色在消隐期和有效区-非图像区不一样？**

这是故意设计的。`de=0` → 黑（符合 HDMI 规范）；`de=1, image_area=0` → 背景色（用户可见）；`de=1, image_area=1` → 图像像素。三个区域三种行为，完整演示了 `de` 信号和 `image_area` 信号的分层控制关系。

**Q3: ROM 读地址为什么只在 image_area 时更新？**

因为背景区不需要 ROM 数据。让 `rom_addr` 在背景区保持为 0 可以减少不必要的 ROM 翻转，略微降低功耗。更重要的是，`image_addr` 在非图像区无意义（`image_x`/`image_y` 被拉到 0），不更新 rom_addr 避免了无效读取。
